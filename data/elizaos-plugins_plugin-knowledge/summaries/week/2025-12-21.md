### Enhanced API Rate Limiting
A new feature was merged to add configurable rate limits for high-throughput APIs, enhancing system stability under heavy load. This update introduces settings for `MAX_CONCURRENT_REQUESTS`, `REQUESTS_PER_MINUTE`, `TOKENS_PER_MINUTE`, and `BATCH_DELAY_MS`, along with an option to disable rate limiting entirely. As part of this change, the default limits for high-throughput APIs were increased fivefold to better accommodate demanding use cases ([#49](https://github.com/elizaos-plugins/plugin-knowledge/pull/49)).
