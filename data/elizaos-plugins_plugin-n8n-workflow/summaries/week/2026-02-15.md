A critical fix was deployed to improve how the plugin handles input from Large Language Models, specifically regarding node property evaluation.
*   **Property Visibility Fix:** Resolved an issue where property defaults were not applied before visibility checks. This ensures that essential fields (like `resource` or `operation`) are correctly evaluated even if the LLM omits them, preventing workflow failures during automated node configuration ([#18](https://github.com/elizaos-plugins/plugin-n8n-workflow/pull/18)).
