# ElizaOS Daily Update (Jun 25, 2025)

## OVERVIEW 
Today's development efforts focused on enhancing plugin architecture, integrating Claude Code, and completing the migration to the new API client. Several bug fixes were also implemented to improve user experience and system stability. Overall, the team made significant strides in modularity and functionality, aligning with our mission to create a robust AI agent framework.

## KEY TECHNICAL DEVELOPMENTS

### Plugin Architecture Enhancements
- Transitioned plugins from project-scoped to agent-scoped architecture, allowing for greater flexibility in plugin configuration for different characters ([#5270](https://github.com/elizaos/eliza/pull/5270)).
  
### Integration of Claude Code
- Added a GitHub Actions workflow for Claude Code integration, enhancing our coding capabilities with AI assistance ([#5269](https://github.com/elizaos/eliza/pull/5269)).

### Migration to New API Client
- Completed the migration of the `packages/client` to the new `@elizaos/api-client`, streamlining our codebase and improving API interactions ([#5263](https://github.com/elizaos/eliza/pull/5263)).

### Bug Fixes
- Updated API documentation for accuracy and added a migration guide ([#5280](https://github.com/elizaos/eliza/pull/5280)).
- Restored functionality for GUI chat title generation, addressing a regression issue ([#5275](https://github.com/elizaos/eliza/pull/5275)).
- Resolved a chat stuck issue that occurred under specific error conditions ([#5274](https://github.com/elizaos/eliza/pull/5274)).
- Improved attachment formatting in the core package to ensure correct handling of message attachments ([#5273](https://github.com/elizaos/eliza/pull/5273)).

## CLOSED ISSUES

### Resolved Bugs
- Fixed the issue of "No handler found for delegate type: TEXT_EMBEDDING," ensuring proper functionality in the system ([#5279](https://github.com/elizaos/eliza/issues/5279)).

## NEW ISSUES

### Plugin and Character Management Challenges
- Reported an issue where users cannot save large character files, resulting in a 500 error due to request entity size limitations ([#5268](https://github.com/elizaos/eliza/issues/5268)).
